# Product Image Generation

当用户提出生成图片相关需求时，严格按下面流程执行。不要新写编排脚本，不要把 skill 安装到全局；只调用项目已有脚本：

```bash
uv run chat "提示词"
uv run image "英文生图提示词" --image-file "参考图路径"
```

## Workflow

1. 读取素材目录，不要猜：
   - 产品和颜色来自 `source/产品/<产品名>/<颜色名>/`
   - 产品描述来自 `source/产品/<产品名>.md`
   - 模特图片来自 `source/模特/<模特名>.<jpg|jpeg|png|webp>`
   - 模特描述来自 `source/模特/<模特名>.md`
   - 参考图来自 `source/参考图/`

2. 产品和颜色识别：
   - 把“用户刚说的原话”和 `source/产品` 下的产品/颜色目录列表发送给 `uv run chat`。
   - 要求 chat 只返回 JSON，例如 `{"matched": true, "product": "...", "color": "..."}`。
   - 如果无法匹配产品或颜色，暂停并用中文问用户选择；先列产品，确定产品后列该产品颜色。
   - 如果只匹配到产品没匹配到颜色，只问颜色。

3. 模特识别：
   - 把“用户刚说的原话”和 `source/模特` 下可用模特列表发送给 `uv run chat`。
   - 要求 chat 只返回 JSON，例如 `{"matched": true, "model": "..."}`。
   - 如果无法匹配，暂停并用中文列出模特让用户选择。

4. 参考图风格反推：
   - 读取 `source/参考图` 下所有图片。
   - 在把参考图发送给 `uv run chat` 前，必须先在本地检查每张参考图的文件大小和像素尺寸：
     - 文件大小超过 800KB，或宽/高任一边超过 1600px，都必须先用电脑上的 `ffmpeg` 压缩/缩放。
     - 不要覆盖原图；把压缩后的图片放到 `source/参考图/_compressed/`，文件名沿用原名但统一输出为 `.jpg`。
     - 压缩命令参考：`ffmpeg -y -i "原图路径" -vf "scale='min(1600,iw)':'min(1600,ih)':force_original_aspect_ratio=decrease" -q:v 5 "source/参考图/_compressed/原文件名.jpg"`。
     - 如果压缩后仍超过 800KB，逐步提高压缩力度（例如 `-q:v 7`、`-q:v 9`）直到不超过 800KB，同时保持最长边不超过 1600px。
     - 后续风格反推优先使用压缩后的文件；未触发压缩条件的参考图继续使用原文件。
   - 如果存在多张参考图，按文件名排序最多选取 5 张一起带图调用 `uv run chat`；少于 5 张则全部使用。
   - 让 AI 综合这些参考图，总结一套统一的英文 style-only reverse prompt。
   - 只要风格、摄影、构图、光线、色彩、质感、后期调性；不要完全复刻参考图，不要绑定具体人物、产品或固定物件。
   - 这一步可以和产品/颜色识别并行。

5. 确定生成张数并生成英文生图提示词：
   - 默认生成 9 张。
   - `main.py` 只能通过环境变量 `IMAGE_GENERATION_COUNT` 定义生成数量；不要根据用户自然语言自行解析张数。
   - 如果需要生成非默认数量，运行脚本前先设置 `IMAGE_GENERATION_COUNT` 为对应正整数。
   - 带上第 4 步的风格反推提示词。
   - 带上所选产品颜色的正面图，例如 `source/产品/MYCENA tote/波子绿/正面.jpg`；如果扩展名不是 jpg，使用实际存在的 `正面.*`。
   - 带上所选模特图片，例如 `source/模特/小乔.jpg`。
   - 带上产品描述 `.md` 和模特描述 `.md` 的文本内容。
   - 必须从产品描述中明确提取并写入产品长宽高；必须从模特描述中明确提取并写入人的身高。如果已知尺寸缺失，不要猜，暂停并要求先补全描述文档。
   - 调用 `uv run chat`，要求只返回 JSON 数组，数组里提示词数量必须正好等于最终生成张数。
   - 所有提示词要都符合参考图调性，但场景、姿态、构图或创意方向各不相同；必须使用指定产品和模特，并保持产品颜色和关键设计细节。
   - 每条英文提示词必须逐字包含明确尺寸参考：人的身高、产品长宽高，并强调严格遵循这些尺寸来确定产品在模特身上的真实比例，避免产品过小或过大。
   - 每条英文提示词必须强调：仔细理解并参考产品图片，保持产品外形、结构、比例、颜色、材质、五金、缝线、肩带/提手、开口等可见细节；不要无端猜想、简化、重设计或修改产品轮廓和结构。
   - 每条英文提示词必须强调：透视关系要正确，模特、产品、手部/身体接触点、镜头角度、阴影、反射和场景深度要自然一致，避免产品漂浮、贴合错误或透视错位。
   - 每条英文提示词必须强调：不要添加海报式文字、标题字、宣传语、字幕、水印、UI 或装饰性文字覆盖层；场景中自然存在的文字或产品本身已有文字可以出现。

6. 生图：
   - 对第 5 步得到的每条英文提示词，分别调用一次 `uv run image`。
   - 每次只生成 1 张图；不要用一次请求生成多张。
   - 每次都带上 1 张模特图，以及所选产品颜色目录下的所有产品图。
   - 实际提交给 `uv run image` 的提示词也必须包含第 5 步的明确尺寸参考（人的身高、产品长宽高）、严格遵循尺寸比例、产品结构、透视关系和禁止海报式文字约束。
   - 可以按最终生成张数并行发起请求。
   - 输出保持在项目现有 `output/` 目录，并按本次运行时间创建子文件夹；图片文件名只带 `task_id`（多图时可加序号），不要再把日期写进图片文件名。

7. 收尾：
   - 用中文告诉用户已选择的产品、颜色、模特，以及生成文件保存位置。
   - 不要运行 test、lint、diff、clean，除非用户明确要求。
