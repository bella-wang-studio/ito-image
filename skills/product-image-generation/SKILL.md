# Product Image Generation

当用户提出生成图片相关需求时，严格按下面流程执行。不要新写编排脚本，只调用项目已有的 `chat` / `image` 脚本：

```bash
uv run chat "提示词"
uv run image "英文生图提示词" --image-file "参考图路径" --output-dir "output/<本次运行时间>/"
```

## Workflow

1. 读取素材目录，不要猜：
   - 产品和颜色来自 `source/产品/<产品名>/<颜色名>/`
   - 产品描述来自 `source/产品/<产品名>.md`
   - 模特图片来自 `source/模特/<模特名>.<jpg|jpeg|png|webp>`
   - 模特描述来自 `source/模特/<模特名>.md`
   - 参考图来自 `source/参考图/`

2. 本地识别产品、颜色和模特：
   - 只读取本地素材文件和目录来识别，不要调用 `uv run chat` 或其他 AI 请求做匹配。
   - 产品候选来自 `source/产品/<产品名>/`，颜色候选来自 `source/产品/<产品名>/<颜色名>/`，产品描述来自 `source/产品/<产品名>.md`。
   - 模特候选来自 `source/模特/<模特名>.<jpg|jpeg|png|webp>` 和同名 `source/模特/<模特名>.md`。
   - 用“用户刚说的原话”在本地候选中做确定性匹配：优先匹配完整产品名、颜色名、模特名；再匹配去空格、大小写不敏感后的名称包含关系。
   - 如果只匹配到 1 个产品、1 个颜色、1 个模特，直接使用这些素材继续。
   - 如果无法匹配产品或颜色，暂停并用中文问用户选择；先列产品，确定产品后列该产品颜色。
   - 如果只匹配到产品没匹配到颜色，只问颜色。
   - 如果无法匹配模特，暂停并用中文列出模特让用户选择。
   - 如果出现多个本地候选都匹配，不要猜，暂停并用中文让用户从候选中选择。

3. 参考图风格反推：
   - 读取 `source/参考图` 下所有图片。
   - 在把参考图发送给 `uv run chat` 前，必须先在本地检查每张参考图的文件大小和像素尺寸：
     - 文件大小超过 800KB，或宽/高任一边超过 1600px，都必须先用电脑上的 `ffmpeg` 压缩/缩放。
     - 不要覆盖原图；把压缩后的图片放到 `source/参考图/_compressed/`，文件名沿用原名但统一输出为 `.jpg`。
     - 压缩命令参考：`ffmpeg -y -i "原图路径" -vf "scale='min(1600,iw)':'min(1600,ih)':force_original_aspect_ratio=decrease" -q:v 5 "source/参考图/_compressed/原文件名.jpg"`。
     - 如果压缩后仍超过 800KB，逐步提高压缩力度（例如 `-q:v 7`、`-q:v 9`）直到不超过 800KB，同时保持最长边不超过 1600px。
     - 后续风格反推优先使用压缩后的文件；未触发压缩条件的参考图继续使用原文件。
   - 如果存在多张参考图，按文件名排序最多选取 5 张一起带图调用 `uv run chat`；少于 5 张则全部使用。
   - 让 AI 综合这些参考图，总结一套统一的英文 style-only reverse prompt。
   - 只要风格、摄影、构图、光线、色彩、质感、后期调性；不要完全复刻参考图，不要绑定具体人物、产品或固定物件。
   - 这一步可以和第 2 步的本地素材识别并行。

4. 确定生成张数并生成英文服装提示词：
   - 默认生成 9 张。
   - 如果用户明确要求生成数量，按用户要求生成对应张数；否则生成 9 张。
   - 根据用户刚说的原话、所选模特描述、参考图信息，以及第 3 步的 style-only reverse prompt，设计同样数量的英文 outfit prompts。
   - 调用 `uv run chat`，要求只返回 JSON 数组，数组里服装提示词数量必须正好等于最终生成张数。
   - 每条服装提示词对应下一步的一张图，必须具体说明模特穿着的完整服装，包括上装、下装或连衣裙/套装、外套、鞋履、主要材质、版型、颜色和必要配饰。
   - 服装必须符合第 3 步参考图调性，并适配模特信息；如果用户明确指定服装风格、单品、颜色或禁忌，必须优先遵循。
   - 服装提示词之间要有明显差异，避免 9 套只做轻微颜色变化；但整体仍应属于同一商业大片/穿搭系列。
   - 不要在服装提示词中新增会和指定产品竞争或混淆的包袋、手袋、托特包、钱包、旅行箱等同类商品；除非用户明确要求。
   - 不要让服装、外套、围巾、头发或大件配饰遮挡产品主体；应让产品佩戴/手持/摆放关系清楚自然。
   - 不要添加带可读广告语、海报文字、品牌大 logo 或图形文字覆盖层的服装。

5. 根据服装提示词生成英文生图提示词：
   - 带上第 3 步的风格反推提示词。
   - 带上第 4 步生成的英文服装提示词，并让每条英文生图提示词按顺序绑定其中 1 套服装；最终第 N 条生图提示词必须包含第 N 条服装提示词的服装设定。
   - 带上所选产品颜色的正面图，例如 `source/产品/MYCENA tote/波子绿/正面.jpg`；如果扩展名不是 jpg，使用实际存在的 `正面.*`。
   - 带上所选模特图片，例如 `source/模特/小乔.jpg`。
   - 带上产品描述 `.md` 和模特描述 `.md` 的文本内容。
   - 必须从产品描述中明确提取并写入产品长宽高；必须从模特描述中明确提取并写入人的身高。如果已知尺寸缺失，不要猜，暂停并要求先补全描述文档。
   - 调用 `uv run chat`，要求只返回 JSON 数组，数组里提示词数量必须正好等于最终生成张数。
   - 所有提示词要都符合参考图调性和对应服装设定，但场景、姿态、构图或创意方向各不相同；必须使用指定产品和模特，并保持产品颜色和关键设计细节。
   - 每条英文提示词必须明确写入模特穿着对应的服装提示词，不要把服装变成可选建议，也不要在最终提示词中改写成另一套衣服。
   - 每条英文提示词必须逐字包含明确尺寸参考：人的身高、产品长宽高，并强调严格遵循这些尺寸来确定产品在模特身上的真实比例，避免产品过小或过大。
   - 每条英文提示词必须强调：仔细理解并参考产品图片，保持产品外形、结构、比例、颜色、材质、五金、缝线、肩带/提手、开口等可见细节；不要无端猜想、简化、重设计或修改产品轮廓和结构。
   - 每条英文提示词必须强调：透视关系要正确，模特、产品、手部/身体接触点、镜头角度、阴影、反射和场景深度要自然一致，避免产品漂浮、贴合错误或透视错位。
   - 每条英文提示词必须强调：不要添加海报式文字、标题字、宣传语、字幕、水印、UI 或装饰性文字覆盖层；场景中自然存在的文字或产品本身已有文字可以出现。

6. 生图：
   - 开始提交生图任务前，先在项目现有 `output/` 目录下按本次运行时间创建 1 个子文件夹，例如 `output/20260523-153012/`。
   - 可用命令创建本次共享目录，例如 `RUN_DIR="output/$(date +%Y%m%d-%H%M%S)"` 后执行 `mkdir -p "$RUN_DIR"`。
   - 对第 5 步得到的每条英文提示词，分别调用一次 `uv run image`，并且每次都传入同一个 `--output-dir "output/<本次运行时间>/"`。
   - 每次只生成 1 张图；不要用一次请求生成多张。
   - 每次都带上 1 张模特图，以及所选产品颜色目录下的所有产品图。
   - 实际提交给 `uv run image` 的提示词也必须包含第 5 步的对应服装提示词、明确尺寸参考（人的身高、产品长宽高）、严格遵循尺寸比例、产品结构、透视关系和禁止海报式文字约束。
   - 可以按最终生成张数并行发起请求。
   - 如果某次 `uv run image` 因网络错误、接口错误、超时、下载失败或其他原因失败，不要排查、不要重试、不要改参数继续尝试；记录/告知该张失败即可，继续等待或执行其他图片任务。这一批成功几张就算几张。
   - 一次批量生成只使用这一个 `output/<本次运行时间>/` 子文件夹；本次生成的所有图片都必须保存到这个同一个文件夹里。例如生成 9 张图，就是同一个文件夹里 9 张图，不是一张图一个文件夹。
   - 图片文件名优先从下载 URL 的文件名里截取 `task_` 后面的部分，并去掉前面的冗余前缀。例如下载 URL 里是 `...gpt_image_2_backup_task_01KSA4B6XHMHRP3Q9FRBCEC88W_0.png`，本地文件名应为 `01KSA4B6XHMHRP3Q9FRBCEC88W_0.png`；多图依次为 `01KSA4B6XHMHRP3Q9FRBCEC88W_1.png` 等。不要把日期、`task_` 或 URL 里的其他备份前缀写进图片文件名。

7. 收尾：
   - 用中文告诉用户已选择的产品、颜色、模特，以及生成文件保存位置。
   - 不要运行 test、lint、diff、clean，除非用户明确要求。
